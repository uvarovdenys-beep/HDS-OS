# IDEA-ARCH-001: Відключаємі модулі захисту (Microkernel Architecture)

## 🚨 Опис проблеми (Ризик переускладнення)
Система набула потужних механізмів захисту та функціоналу (AST Validation, Checksums / Knowledge Gatekeeper, Browser Agent, AI Vision, Token Wallet). 
Якщо жорстко "вшити" ці важкі перевірки безпосередньо в основний цикл `agent.py` або `orchestrator.py`, виникнуть критичні проблеми:
1. **Катастрофічне падіння швидкості**: Постійні прорахунки SHA-256 хешів та побудова дерев AST гальмуватимуть обробку базових скриптів.
2. **Складність дебагінгу**: При помилці буде неможливо швидко зрозуміти причину (чи це Gatekeeper заблокував доступ, чи AST знайшов хибне спрацьовування, чи скрипт впав сам по собі).
3. **Монолітність**: Падіння одного з важких модулів (наприклад, Browser Agent) може потягнути за собою "смерть" всього ядра HDS.

## 💡 Пропоноване рішення: Плагінна Екосистема
Необхідно перевести HDS на архітектуру "Тонкого Ядра" (Thin Core / Microkernel):
1. **Мінімалістичний `agent.py`**: Ядро має відповідати ВИКЛЮЧНО за читання черги тасків, базову маршрутизацію і виклики `VoxService`. Жодної бізнес-логіки валідації.
2. **Опціональні Middleware (Плагіни)**: Всі механізми безпеки мають бути винесені в незалежні модулі, які підключаються за потреби.
3. **Гнучка конфігурація (`hds_config.yaml`)**:
   Впровадити конфіг-файл, який дозволить включати/вимикати "важку броню":
   ```yaml
   core:
     strict_sandbox: true
   security_plugins:
     ast_validation_enabled: false   # Вимкнено для швидкодії, вмикається лише для підозрілих тасок
     gatekeeper_checksums: true
   modules:
     browser_daemon: true
     vision_api: false
   ```
4. **Ізоляція процесів (IPC)**: Важкі модулі (як браузер або візія) повинні працювати як окремі демони. Ядро просто надсилає їм локальні HTTP/Socket запити.

## 🎯 Очікуваний результат
Система збереже статус "Непробивної фортеці" (Zero-Trust), але при цьому залишиться швидкою, гнучкою та неймовірно легкою в обслуговуванні. Якщо "впаде" плагін браузера — ядро ОС цього навіть не помітить і просто перейде до наступної таски.

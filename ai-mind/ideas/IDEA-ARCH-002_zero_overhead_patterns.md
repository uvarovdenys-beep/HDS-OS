# IDEA-ARCH-002: Zero-Overhead OS Patterns

## Опис
Для забезпечення стабільності та можливості побудови складних пайплайнів без завантаження Ядра (CPU), пропонується впровадити чотири файлово-орієнтовані механізми:

1. **Dead Letter Queue (DLQ)**
   - **Суть**: Якщо скрипт падає з помилкою, він переноситься з `tasks/active/` до `tasks/failed/` разом зі згенерованим `.json` звітом про помилку (Stack Trace).
   - **Перевага**: Система не намагається виконувати "зламане" завдання в нескінченному циклі, лоб економлячи ресурси.

2. **Task Dependencies (DAG)**
   - **Суть**: Мета-тег `depends_on: ["TKT-123"]` у файлі задачі.
   - **Перевага**: Ядро пропускає виконання задачі, доки залежна задача не з'явиться у папці `archive`. Дозволяє будувати складні багатоетапні алгоритми.

3. **Cron-like Scheduling**
   - **Суть**: Мета-тег `run_after: <timestamp>`. Ядро ігнорує файл до настання цього часу.
   - **Перевага**: 0% навантаження CPU на "чекання". Дозволяє ставити таски на ніч.

4. **Маніфест залежностей (Pre-flight checks)**
   - **Суть**: Перевірка необхідних бібліотек Python перед запуском скрипта. Якщо пакета немає -> таска йде в DLQ. Захищає від `ImportError` під час рантайму.

## Наступні кроки
Інтегрувати парсинг цих мета-тегів безпосередньо у функцію читання завдань Ядра (`agent.py`).

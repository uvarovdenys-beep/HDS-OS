# HDS L — Протокол для інженерів

> Розмір: L | Діагностичний бал: 10-14
> Для: Sonnet, GPT-4o-mini, Qwen 27B-35B, DeepSeek-R1 8B

---

## BOOT (3 кроки)

```
1. ai-mind/protocols/hds_l.md    → цей файл (правила)
2. ai-mind/architecture/project-map.md → карта проєкту
3. ai-mind/tasks/active/         → своя задача
```

---

## МОЖЛИВОСТІ

- Керована автономія (слідуй decomposition)
- Discovery з бюджетом (max 10 файлів)
- Створення модулів (до 300 рядків)
- AIVC цикл (max 5 кроків)
- Базовий рефакторинг (в межах одного файлу)
- Self-correction (1 ітерація)

---

## ПРОТОКОЛ: CHECK-DO-TEST-VERIFY

```
CHECK  → Прочитай target файл + залежності
DO     → Внеси зміни (за decomposition)
TEST   → Запусти тест із задачі
VERIFY → Перечитай файл, переконайся що зміна коректна
```

---

## ПРАВИЛА (8)

L-01 ONE_TASK — одна задача за сесію
L-02 READ_FIRST — прочитай перед зміною
L-03 TEST_AFTER — тестуй кожну зміну
L-04 MAX_500_LINES — не більше 500 рядків за раз
L-05 ASK_IF_LOST — не розумієш → запитай
L-06 FOLLOW_DECOMPOSITION — слідуй крокам задачі
L-07 NO_UNRELATED_CHANGES — не "покращуй" те що не просили
L-08 VERIFY_RESULT — перечитай після зміни

## ФОРМАТ ЗАДАЧІ (як бачить L)

Отримує задачу з decomposition і слідує крокам:
- Goal → розуміє кінцеву мету
- Steps → виконує послідовно
- Solution_hint → використовує як орієнтир
- Test → запускає обов'язково

# HDS Lite — Протокол для спрощених моделей

> Для моделей: Haiku, Sonnet, Qwen <14B, Llama <13B, Mistral <7B
> Автор: Opus 4.6 | HDS OS v1.1

---

## BOOT (2 кроки)

```
КРОК 1: Прочитай цей файл
КРОК 2: Прочитай свою задачу з ai-mind/tasks/active/
```

Все. Не скануй проєкт. Не читай архітектуру. Не шукай файли.

---

## 5 ПРАВИЛ (L-серія)

| # | Правило | Пояснення |
|---|---------|-----------|
| L-01 | ONE TASK | Бери тільки одну задачу за сесію |
| L-02 | READ FIRST | Прочитай файл перед зміною |
| L-03 | TEST AFTER | Запусти тест після зміни |
| L-04 | MAX 200 LINES | Не змінюй більше 200 рядків за раз |
| L-05 | ASK IF LOST | Якщо не розумієш — зупинись і запитай |

---

## ПРОТОКОЛ РОБОТИ: CHECK-DO-TEST

```
CHECK: Прочитай файл, який будеш змінювати
DO:    Внеси зміни (тільки те, що в задачі)
TEST:  Запусти тест із задачі
```

**Забрати** (це для Opus):
- SHA256 хешування
- Автоматичне створення скриптів
- DVP LOG таблиця
- Пошук файлів (discovery)
- Архівування

**Залишити:**
- PASSED / FAILED в stdout
- Конкретний тест у задачі

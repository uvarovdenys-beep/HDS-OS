# HDS Permission Matrix — Що ЗАБОРОНЕНО кожному розміру

> Протокол = не рекомендація. Протокол = ACL.
> Модель читає свої заборони при BOOT і не може їх обійти.

---

## Принцип

```
Протокол каже моделі не "що ти МОЖЕШ" а "що тобі ЗАБОРОНЕНО".
Все що не заборонено — дозволено.
Все що заборонено — модель НЕ ПОВИННА навіть НАМАГАТИСЬ.
```

## S — Помічник

### ❌ ЗАБОРОНЕНО:

| Дія | Чому заборонено |
|-----|----------------|
| Перетворити ідею в задачі | Не бачить архітектуру, створить сміття |
| Створювати задачі для інших | Не має контексту системи |
| Шукати файли в проєкті | Витратить весь контекст, забуде задачу |
| Створювати нові модулі | Не розуміє де і як вони мають працювати |
| Рефакторити код | Зламає залежності яких не бачить |
| Працювати по DVP протоколу | Загубиться на кроці 2 з 5 |
| Приймати архітектурні рішення | Не має картини системи |
| Запускати AIVC цикл | Не вміє планувати і адаптуватись |
| Змінювати >50 рядків | Не може тримати контекст великої зміни |
| Читати >3 файли | Забуде перші коли прочитає треті |
| Виправляти свої помилки | Зробить гірше (зациклиться або зламає інше) |
| Оптимізувати код | Перепише робоче на неробоче |
| Видаляти код | Не розуміє чому він там |

### ✅ ДОЗВОЛЕНО:

- Відкрити файл → внести зміну із задачі → запустити тест
- Скопіювати код із задачі точно як є
- Показати результат тесту
- Сказати "не розумію" і зупинитись

- Виправляти прості помилки (typo, missing import)
- Створювати задачі для S (прості, конкретні)
- Сказати "не розумію" і зупинитись

## Як це працює технічно

1. **BOOT**: Модель читає СВІЙ протокол (hds_s.md / hds_m.md / hds_l.md / hds_xl.md)
2. **У протоколі**: Секція "ЗАБОРОНЕНО" — перше що читає
3. **Задача приходить**: Роутер перевіряє `task.complexity` vs `model.max_task_complexity`
4. **Якщо задача занадто складна**: Роутер відхиляє і маршрутизує на вищий рівень
5. **Якщо модель намагається порушити**: Протокол вже в контексті — модель "знає" що не можна

## Безпека: Що якщо модель порушить заборону?

**Превентивно:**
- Протокол — частина system prompt, модель "знає" заборони
- Задача адаптується під розмір (S навіть не бачить поле decomposition)
- Роутер НЕ ДАСТЬ задачу з complexity > max_task_complexity

**Детективно:**
- Webhook server перевіряє source: якщо M-модель пробує POST задачу → відхилити
- Agent.py перевіряє: якщо зміна > max_lines_per_change → rollback
- Git diff: якщо торкнулось файлів поза target → alert

**Реактивно:**
- XL модель (verify pass) перевіряє результати нижчих рівнів
- Якщо M зламала щось → задача переходить на L
- Якщо L не справилась → ескалація на XL
